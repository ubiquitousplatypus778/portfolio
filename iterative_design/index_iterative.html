Iterative Design

for Homeroom

Project Type

Group project

Strategies

Iterative Prototyping, User Testing

Tools Used

Balsamiq, Figma

Duration

2 Weeks

Overview

I worked in a team of four to iteratively design a web application for a startup ---- Homeroom, which helps young people living in cities that lack affordable housing options rent single rooms and helps investors renovate their homes and find qualified renters.

Main Takeaway

The cyclic process of prototyping, testing, analyzing, and refining a product allows for greater efficiency and timely revisions. With each round of iterative prototyping, our team was able to continually improve the product. What’s more, user testing enables an objective assessment.

Step 1: Sketching & Wireframing

For this assignment, we decided to design a desktop website for the startup. We thought that the website would make it easier for investors to find and connect with the startup, as well as help young people find a room to rent.

We started by quickly sketching out many different versions of the webpage.

Step 4: Mockup Critique

We presented our prototype in a mockup critique studio, during which we briefly introduced our Homeroom Concept and let our peers interact with our prototype. We then received feedback from the TAs, other students and our industry guest Gifford Cheung, a UX Researcher at Nintendo. We took notes of the feedback and made a final iteration to our prototype.

Feedback Summary

  • Need more info about the properties.
  • Button labels need more specification.
  • Learnability of UI is lacking. For example, it is not clear from the search bar on the first page what the user is searching for. Should offer more guidance to the user.
  • Need an option to message between renter and investor because it involves the transaction of a large sum of money.
  • Both a back button and “X” button on some pages and they have different transitions. Make it consistent.
  • Aligned / justified text (money decimal point) is done well.

Step 5: User Testing

Having test users try out an interface is an important part of testing and a valuable source of feedback. We conducted a remote usability test on three users through a remote user testing service (UserTesting.com).

We provided users with a brief introduction of the functionality of our website and two main tasks to carry out:

  • Task 1: You are a renter looking for a room. Find the cheapest room available, add it to your cart, and check out your order.
  • Task 2: You are a wealthy landlord interested in using Homeroom. You have a budget of roughly 1.2 million USD. Find the renovation plan with the largest square footage, and submit a proposal.

Main Results

  • Many problems were caused by users not being able to navigate the overlaying Figma and and UserTesting UIs, which indicates that we should provide more instruction on how to use Figma.
  • Two testers had trouble figuring out the functionality of different buttons. While both users were eventually able to figure this out, the time it took them to do so suggests that we could make this even more learnable in a further iteration.
  • We were expecting testers to directly follow the main tasks as laid out in the scenario, but our first tester focused heavily on the “side tasks,” ignoring the main instructions. We should be more explicit about our expectations of the testers in the instructions.
